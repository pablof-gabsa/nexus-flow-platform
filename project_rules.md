# Reglas del Proyecto Nexus Flow

## Directivas de Trabajo

1.  **Sincronización Continua**:
    -   **SIEMPRE** que se realice un cambio en el código (por pequeño que sea), debe subirse al repositorio inmediatamente.
    -   No dejar cambios locales sin commitear/pushear al finalizar una tarea.

2.  **Estructura del Repositorio**:
    -   La raíz del repositorio contiene directamente los archivos de la aplicación (`index.html`, `js/`, `css/`, etc.).
    -   No crear subcarpetas innecesarias para el proyecto principal.
